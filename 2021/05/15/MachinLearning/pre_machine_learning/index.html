    01 머신 러닝이란?

    프로그램이 스스로 규칙을 찾게함

    톰 미첼 : 기계가 학습한다는 건, 프로그램이 특정 **작업(T)**를 하는 데 있어서 **경험(E)**을 통해 작업의 **성능(P)**을 향상시키는 것

    머신러닝 프로그램의 예시 : 스팸 메일 분류 프로그램

    • 작업 : 스팸 분류
    • 경험 : 새로운 메일 확인하고 분류
    • 성능 : 분류 확률

    머신 러닝이 핫해진 이유

    1. 사용가능한 데이터가 많아짐
      • 경험(E) = 데이터 (옛날엔 아날로그 저장 또는 저장해야 한다는 개념이 없었음)
    2. 컴퓨터 성능이 좋아짐
      • 빠르게 계산, 학습을 시키는데 시간 소요가 옛날보다 적음
    3. 활용성 증명
      • 유튜브 추천 알고리즘 등 $\rightarrow$ 시청률 업
      • 맞춤형 광고 $\rightarrow$ 수익 창출

    인공 지능? 빅 데이터? 머신 러닝?

    • 빅데이터
      • 데이터 보관/ 처리법
      • 데이터 분성 방법들
    • 인공지능
      • 프로그램이 인간처럼 생각/행동하게 하는 학문
      • 머신 러닝은 인공지능의 수단 중 하나
    • 딥 러닝
      • 머신 러닝 기법 중 하나

    학습의 유형

    • 지도학습 (Supervised learning) : 답이 있고 답을 맞추는게 학습의 목적
      • 분류(Classification) : 스팸 매일 분류
      • 회귀(Regression) : 아파트 가격 예측
    • 비지도학습 (Unsupervised learning) : 답이 없고, 이 답을 맞추는게 학습의 목적
      • 알파고
      • 이 기사 분류가 어떤지 맞춰봐 등

    • 강화학습

    k-NN 알고리즘

    kNN(k-Nearest Neighbors Algorithm, k 최근접 이웃 알고리즘)

    예시 : 타이타닉 탑승자 데이터 (답이 있으므로 지도학습, 생존 or 사망으로 분류 문제)


    많은 경험 (데이터가 많으면) $\rightarrow$ 성능 향상 $\rightarrow$ 머신 러닝

    머신 러닝의 수학

    머신 러닝 = 컴공 + 수학

    코드 몇줄이면 사용이 가능 

    from sklearn.linear_model import LogisticRegression

model = LogisticRegression()
model.fit(X_train, Y_train)

    그러나, 알고리즘 선택과 코드 최적화를 위한 수학 지식은 필요함!!!!

    필요한 수학

    선형대수, 미분, 통계, 확률

    • 선형 대수학

      • 행렬 : 값을 직사각형의 형태로 표현
        • 많은 데이터를 묶을 수 있음
        • 계산시 효율적 (개별적으로 계산하지 않고 묶어서 계산)

    • 미적분학

      • 머신 러닝에서 최적화를 위해 사용

    • 통계

      • 데이터의 특징 파악 (하여 예측)
        • 예시(타이타닉) : 생존자 vs 사망자 평균 티켓값, 생존자 vs 사망자 평균 나이

    • 확률

      • 가능성을 공부하는 확률
        • 예시(타이타닉) : 생존확률 70%, 20% 등
