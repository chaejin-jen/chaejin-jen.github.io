    01 선형 회귀 (Linear Regression)

    예시 : 집 크기가 주어졌을 때, 이걸 이용해서 집 값을 예측

    선형 회귀는 여기 있는 데이터를 가장 잘 대변해 주는 선 을 찾아내는 것

    • 이 데이터에 가장 잘 맞는, 가장 적절한 하나의 선을 찾아냄
      • 통계학에선 최적선(line of best fit)이라고 함

    데이터에 잘 맞는 최적선을 찾았다고 가정했을 때, 최적선의 활용

    최적선에 해당하는 집 크기 대비 집 값을 찾아내면 완전히 정확하지는 않더라도 꽤 합리적인 예측이 가능함

    선형 회귀 알고리즘은 단순하면서도 유용하고, 특히 다른 많은 알고리즘의 기반이 되기 때문에 중요

    용어

    • 머신러닝 분류
      • 지도 학습
        • 분류 : 불연속적인 값 (아파트/주택/오피스텔, 찬성/반대)
        • 회귀 : 연속적인 값
      • 비지도 학습
      • 강화 학습
    • -

    선형 학습 은 답을 알려주기 때문에 지도학습 이며, 연속적인 값을 예측하기 때문에 회귀 문제이다.

    변수

    선형 회귀에서 우리는 어떤 정보(집 크기)를 갖고, 어떤 답(집 가격)을 예측한다.

    목표 변수 : 우리가 맞추려고 하는 값 (output)
    (target variable / output variable)

    입력 변수 : 목표 변수를 맞추기 위해서 사용하는 값 (input)
    (input variable / feature) (feature - 특징, 속성)

    예를 들어서 집의 크기를 갖고 가격을 예측하려고 하면

    • 집 크기 = 입력 변수
    • 집 가격 = 목표 변수
      즉, 집 크기라는 속성 을 이용해서 집 가격을 예측

    데이터 표현법

    • 학습 데이터 : 우리가 프로그램을 학습시키기 위해 사용하는 데이터
      • 학습 데이터의 개수 (m)
    • 입력 변수 ($x$)
      • i 번째 입력 변수 : $x^{(i)}$
    • 목표 변수 ($y$)
      • i 번째 목표 변수 : $y^{(i)}$
      • 만약 데이터가 50개 있으면, 이 50개를 각각 어떻게 나타낼 수 있을까요?

    가설 함수

    선형 회귀의 목적 : 데이터에 가장 잘 맞는 최적선 찾고, 그 최적선을 이용해서 새로운 입력 변수에 대한 목표 변수를 예측하는 것

    가설 함수(hypothesis function): 최적선을 찾아내기 위해 시도한 다양한 함수

    • 선형 회귀에서의 최적선 : 직선 (일차 함수)
      • $y = ax + b$
        즉, 계수 a, 상수 b 찾기

    • 선형 회귀의 가설 함수 표현 (입력변수가 여러개)

      • $f = ax + by + cz + dq + er + \cdots$ 복잡

        $$h_\theta(x) = \theta_0 + \theta_1x_1 + \theta_2x_2 + \theta_3x_3 + \cdots \qquad \qquad \qquad$$

      • 선형 회귀의 임무는 가장 적절한 이 세타 값들을 찾아내는 것

      • $x_0 = 1$ (bias)
        : $\theta_0$ 는 상수항이기 때문에 곱해지는 입력 변수가 없지만, 통일성을 위해 다음과 같이 써줌

        $h_\theta(x) = \theta_0x_0 + \theta_1x_1 + \theta_2x_2 + \theta_3x_3 + \cdots \qquad \qquad \qquad$

    가상 함수 평가법

    평균 제곱 오차 (MSE)

    MSE(mean squared error) : 선형 회귀에서 가장 많이 쓰는 방법

    데이터들과 가설 함수가 평균적으로 얼마나 떨어져 있는지 나타내기 위한 하나의 방식 (작을 수록 좋음)

    MSE : 오차 제곱값의 평균

    • 오차 : 예측 값에서 원래 값을 뺀 값
    • 오차값들을 모두 제곱하여 더한 후, 총 데이터 개수 만큼 나눔

    [예시] 입력변수 한개

    아래 그림처럼 데이터가 주어졌을 때, 세가지 가설 함수를 세웠다고 가정해보고, MSE 로 평가해보자.
    가설함수 : $h_\theta(x) = \theta_0 + \theta_1x$

    $MSE: \frac{-3.2^2 + 6.6^2 + \cdots}{14}$

    평균 제곱 오차가 크다 = 가설 함수와 데이터들 간의 오차가 크다

    • $\rightarrow$ 해당 가설 함수는 데이터들을 잘 표현해 내지 못함

    MSE : 11.2 로 가장 작으므로 두번째 가설이 주어진 데이터셋에 가장 잘 맞는 가설이다.

    제곱을 하는 이유

    1. 오차의 부호를 양수로 통일
    2. 오차가 커질수록 더 부각시키기 위함 (더 큰 오차에 대해서는 더 큰 페널티를 줌)

    평균 제곱 오차 일반화

    $$\frac{1}{m}\displaystyle\sum_{i = 1}^{m}(h_\theta(x^{(i)}) - y^{(i)})^2 \qquad \qquad \qquad$$

    뜯어보기

    • $(h_\theta(x^{(i)}) - y^{(i)})^2$
      $h$ : 가설 함수
      $x$ : 인풋 (즉, $x^{(i)}$ 는 i번째 인풋)
      $y$ : 아웃풋 (즉, $y^{(i)}$ 는 i번째 아웃풋)

      1. 가설 함수 $h$에 $x^{(i)}$를 넣으면 $x^{(i)}$에 대한 예측값이 나옴
      2. 그 예측값에 $y^{(i)}$를 빼면, 이 가설 함수가 예측하는 값과 실제 데이터 값의 오차
      3. 그 오차값을 제곱한 것

    • $\displaystyle\sum_{i = 1}^{m}$

      1. i에 1부터 m까지 대입해서 계산한 후, 그 값들을 모두 더함

        ∑ 의미

        $\displaystyle\sum_{i = 1}^{m}$

        i에 1부터 m까지 반복적으로 대입해라
        i에 1을 대입해서 계산하고, i에 2를 대입해서 계산하고, i에 3을 대입해서 계산하고… 이런 식으로 i에 1부터 m까지 대입해서 계산한 후, 그 값들을 모두 더하는 것 까지가 시그마의 역할

    • $\frac{1}{m}$

      1. 마지막에 그 합을 총 개수인 m으로 나눠서 평균을 냄

    손실 함수

    손실 함수(loss function, $J$) : 어떤 가설 함수를 평가하기 위한 함수

    손실 함수의 아웃풋이 작을수록 가설 함수의 손실이 적기 때문에 더 좋은 가설 함수
    반대로 손실 함수의 아웃풋이 클수록 가설 함수의 손실이 큰 거기 때문에 더 나쁜 가설 함수

    선형 회귀의 경우에는 평균 제곱 오차가 손실 함수의 아웃풋
    $$J(\theta) = \frac{1}{2m}\displaystyle\sum_{i = 1}^{m}(h_\theta(x^{(i)}) - y^{(i)})^2$$

    특정 가설 함수의 평균 제곱 오차 $\uparrow$ $\rightarrow$ 이 손실 함수의 아웃풋이 $\uparrow$ $\rightarrow$ 손실이 크기 때문에 안 좋은 가설 함수
    반대로 가설 함수의 평균 제곱 오차 $\downarrow$ $\rightarrow$ 이 손실 함수의 아웃풋 $\downarrow$ $\rightarrow$ 손실이 적기 때문에 좋은 가설 함수

    MSE 에서 $\frac{1}{m}$ 이 $\frac{1}{2m}$ 로 바뀐건 그냥 이후에 계산을 더 편하게 하기 위한 장치 (자세한 설명은…)

    손실 함수 $J$의 인풋 : $\theta$(세타)

    • 보통 $\theta$ 하나로 묶어서 표현
      ($\theta_0$, $\theta_1$, … 이런 식으로 따로따로 써도 됨)

    • 가설 함수 예시 : $h_\theta(x) = \theta_0 + \theta_1x$

      • $\theta$ : 바꿀 수 있는 값 (input)
        즉, 세타 값들을 잘 조율해서, 가장 적합한 가설 함수를 찾아내는 것
        결국 손실 함수의 아웃풋은 이 세타 값들을 어떻게 설정하느냐에 달려 있음
