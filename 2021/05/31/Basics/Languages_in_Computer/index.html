    Programming Language vs Markup Language

    • Programming Language : 컴퓨터에 지시를 내리는 언어
    • Markup Language : 특정 형식과 문법대로 요소나 절차들을 배치하는 언어
      • 각각의 요소들이 어떤식으로 배치되어 시각화 될 것인가 + 검색엔진 등 소프트웨어 시스템이 페이지들을 제대로 분석하게 하기 위해 사용
        • 예를 들어 h6 보단 h2에 검색어가 들어가 았는게 더 알맞은 검색결과
        • 마크업된 검색결과로 서브페이지도 검색 되게 함
      • 서로 다른 시스템과 편집툴을 거쳐도 공용적으로 사용할 수 있도록 문서타입을 만든 것
        • 지식을 공유하기 위한 문서작업이 발단이 됨
      • HTML (HyperText Markup Language)
        - HyperText : 처음부터 끝까지 순서대로 읽어낼 수 있었던 기존 문서를 뛰어넘어서, 중간중간 링크가 들어있어서 이동할 수 있는, 초월문서를 뜻하는 말 (하이퍼링크에서 하이퍼와 같은 뜻)

    - WYSWYG (What You See What You Get) : GUI 환경에서 만드는 웹 (이거 망함)

    모델링 언어

    데이터 모델링 언어 : XML, XrML, JSON

    통합 모델링 언어 : UML

    데이터 모델링 언어

    XML(eXtensible Markup Language)

    : W3C(World Wide Web Consortium)에서 개발되었고, 웹브라우저 간 호환이 되지 않는 문제와 SGML(Standard Generalized Markup Language)의 복잡함을 해결하기 위해 개발된 다목적 마크업 언어

    XrML (eXtensible rights Markup Language)

    : XML기반의 언어로 리스프(LISP)기반의 DRM언어인 디지털 저작권언어(DPRL)를 개발하여 디지털 콘텐츠 / 웹 서비스 권리 조건을 표현
    모든 자원과 관련된 권리 및 조건들을 보안적으로 규정하고 관리하는 일반적인 방법을 제공하여 콘텐츠 및 서비스의 신뢰를 제공

    JSON (JavaScript Object Notation)

    : 비동기 브라우저/서버 통신(AJAX)을 위해 “속성-값 쌍”, “키-값 쌍”으로 이루어진 데이터 오브젝트를 전달하기 위해 인간이 읽을 수 있는 텍스트를 사용하는 개방형 표준 포맷
    Ajax(Asynchronous JavaScript and XML)에서 많이 사용
    XML(eXtensible Markup Language)을 대체하는 주요 데이터 포맷
    언어 독립형 데이터 포맷으로 다양한 데이터 프로그래밍 언어에서 사용

    통합 모델링 언어

    UML (Unified Modeling Language)

    : 객체 기술에 관한 국제표준기구(OMG)에서 정의한 표준으로 시스템 분석, 설계, 구현 등 개발자와 사용자 간 의사소통이 원활하게 이루어지도록 표준화한 객체 지향 모델링 언어를 뜻하는 개념
    구성요소(3): 사물 / 관계 / 다이어그램(=도안 표기법이 포함됨)

    프로그래밍 언어

    프로그래밍 패러다임 : Procedural Programming, Imperative programming($\leftrightarrow$ Declarative programming), Functional programming, Logic programming

    Statically typed languages : 컴파일 타임에 변수 유형이 알려진 경우 (C, C++, Java, Rust, Go, Scala)

    Dynamically typed languages : 유형이 이름이 지정된 변수 / 필드 / 등이 아니라 런타임 값과 연관되는 경우 (Perl, Ruby, Python, PHP, JavaScript)

    플랫폼, 독립적인, 인터프리터 언어

    • 독립적인 = 관계없는, 상관없는
      : OS에 상관없이 한번 프로그램을 작성하면 사용 가능

    • 플랫폼 = OS
      : 윈도우, 리눅스, 안드로이드, 맥OS, iOS 등 프로그램이 실행되는 운영체제를 플랫폼 이라고 함

    • 인터프리터 = 통역기를 사용하는 언어…
      : 소스코드를 바로 실행할 수 있게 지원하는 프로그램 실행 방법

    돔(DOM, Document Object Model)

    : 웹페이지가는 브라우저상에서 파싱(Parsing)되어 표현되는데, 그 표준 형식

    • 돔 조작이 발생할 때마다 배치나 화면 표시에 많은 연산을 발생하므로 조작이 빈번해질수록 브라우저 성능이 낮아지는 문제 발생 $\rightarrow$ 가상 돔 채용 ($\rightarrow$ React, Vue.js 등)
