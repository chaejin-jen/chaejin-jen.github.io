    Promise

    • 자바스크립트 비동기 처리에 사용되는 객체
    • 서버에서 받은 데이터를 화면에 표시할 때 사용하며 웹 구현시 다음과 같은 API사용
      $.get('url주소/products/1', function(response){
    // ...
});

    사용 이유

    빠른 처리 + 순차 시행 + 깔끔한 코드 작성

    • 싱글쓰레드인 자바스크립트에서 비동기 처리를 위함
    • 즉, 순차 시행이 필요한 경우 빨리하려고!

    예시 : 1번 제품을 받아오고, 3번 제품을 받아오고,…. response 내용 뿌리기

    1. blocking : 단계적 작성 작성하기 좋으나 순서대로 프로그램이 동작하므로 성능이 떨어짐

      exports.get_products_detail = ( req , res ) => {
    const product1 = models.Products.findByPk(req.params.id).then( (product) => {
    });

    const product2 = models.Products.findByPk(req.params.id).then( (product) => {
    });
    
    res.render('admin/detail.html', { product1 : product1, product2 : product2 });  
};

    2. callbackhell : 늘어날 수 록 가독성이 떨어짐 $\rightarrow$ 유지보수 어려움

      • callback 함수가 결과값을 가지고 callback을 다시 호출하고 이 과정을 인라인으로 처리하며 코드가 복잡하지는 것을 말함

      • 해결방법

        1. 인라인 함수에 이름 붙이기
        2. 코드 간결하게 작성하기
        3. 모듈화
        4. Promise 도입

          1, 2, 3 은 한계점이 존재하므로, Promise를 도입하자!

    우리가 원하는 것 : 순차적으로 적혀있는 코드지만, 동시실행 $\rightarrow$ Promise 사용이유

    Promise 만들기

    1. resolve 와 reject를 인자로 가지는 함수를 만든다.

      • Promise 객체 생성 :
        const p1 = new Promise( (resolve , reject) => { } )

    2. 내가 원하는 시점을 resolve로 받아오자.

    [참고] Promise 선언부

    • ‘지금은 없는데 별 문제 없으면 주고 없으면 알려줄게’ 라는 약속
      1. pending : 약속을 수행중인 상태
      2. fulfilled : 약속이 지켜진 상태
      3. rejeted : 약속이 어떤 이유에 의해 못 지켜진 상태
      4. settled : 약속이 이행됬든 안됬든 결론이 난 상태

    resolve 예시

    • 기본

      const aa = new Promise(( resolve, reject ) => {
    resolve( console.log('프라미스 이행!!!') );
});

aa.then(() => {
    console.log("프라미스 실행 완료!!!");
});

    • 결과를 받아와 보자

      const aa = new Promise(( resolve, reject ) => {
    resolve( "promise 123" );
});

aa.then((result) => {
    console.log(result);
});

    • 1초뒤 실행

      const wait1second = new Promise(( resolve, reject ) => {
    console.log("promise 시작!!");
    setTimeout( () => {
        resolve( console.log("1초뒤에 찍습니다!!") );
    } , 1000);
    
});

wait1second.then(() => {
    console.log("promise 이행완료!!");
});

    reject 예시

    • 1초뒤 실행

      const wait1second = new Promise(( resolve, reject ) => {

        reject( console.log("에러!!!") );

});

    console.log("promise 이행완료!!");
});

    • [에러처리] catch

      const wait1second = new Promise(( resolve, reject ) => {
        reject("에러!!!");
    
});

wait1second.then(() => {
    console.log("promise 이행완료!!");
}).catch( (err) =>{
    console.log(err);
});

    • [활용예시] 로그인 인증 (안돌아가는 코드)

      const login = new Promise(( resolve, reject ) => {
    
    if(){
        resolve(console.log("로그인성공!!"));
    }else{
        reject( console.log("로그인실패!!!") );
    }
});

login.then(() => {
    console.log("promise 이행완료!!");
});

    Promise Chaining / Promise all

    • Promise Chaining
      • 각자 따로따로 실행하는 분기점을 가지면서도, 순차적인 진행 가능
    • Promise all
      • p1, p2 가 Promise 객체 일때 Promise all 문법 :
        Promise.all([ p1,p2 ]).then((result)=>{})
    promise 적용 전 promise 적용 후
    순차적으로 2초, 3초 걸려서 실행하므로 5초가 소요됨 각자 따로 분기점을 갖고 실행하며, 최종적으로 끝나는 애에 맞춰서 실행 함 (끝나는 시간 보장) 3초가 소요됨
