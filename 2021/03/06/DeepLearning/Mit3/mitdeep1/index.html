    4.1 미분이란

    4.2 수치 미분 구현, 합성 함수의 미분

    아래 식에 따라 미분을 계산하는 코드를 구현

    • 조건1 : 컴퓨터에선 극한을 취급할 수 없으므로, h를 극한과 비슷한 값으로 대체 (ex: $h=0.0001$) $\rightarrow$ 수치미분!

      • 수치미분 (numerical differentiation) : $h=1e-4$ 와 같은 미세한 차이를 이용하여 함수의 변화량을 구하는 방법

        • 실제 미분값에 근접함
        • 전진차분(foward difference) : 우리가 생각하는 일반적인 수치미분
          • $f(x)$ 와 $f(x+h)$의 차이를 구함
        • 중앙차분 (centered difference) : 근사 오차를 줄이는 방법으로 사용 됨
          • $f(x-h)$ 와 $f(x+h)$의 차이를 구함
          • 기울기 : $\frac{f(x+h) - f(x-h)}{2h}$
          • 근사 오차가 적다는 것은 테일러급수(Taylor series)를 이용해 증명 가능

    import numpy as np

# 1
class Variable:
    def __init__(self, data):
        self.data = data

# 2
class Function:
    def __call__(self, input):
        x = input.data
        y = self.forward(x)
        output = Variable(y)
        self.input = input
        self.output = output
        return output

    def forward(self, x):
        raise NotImplementedError()


class Square(Function):
    def forward(self, x):
        return x ** 2

# 3
class Exp(Function):
    def forward(self, x):
        return np.exp(x)

# 4-1
def numerical_diff(f, x, eps=1e-4):
    x0 = Variable(x.data - eps)
    x1 = Variable(x.data + eps)
    y0 = f(x0)
    y1 = f(x1)
    return (y1.data - y0.data) / (2 * eps)

# 4-2
f = Square()
x = Variable(np.array(2.0))
dy = numerical_diff(f, x)
print(dy)

# 4-3
def f(x):
    A = Square()
    B = Exp()
    C = Square()
    return C(B(A(x)))

x = Variable(np.array(0.5))
dy = numerical_diff(f, x)
print(dy)

    실행 결과

    4.000000000004
3.2974426293330694

    뜯어보기

    1. numerical_diff(f, x, eps=1e-4) 함수

      • f : 미분의 대상이 되는 함수 (Function의 인스턴스)

      • x : 미분을 계산하는 변수 (Variable의 인스턴스)

      • eps : 아주 작은 값 - 함수의 변화량을 수치 미분하기 위한 미세한 값

        • eps 는 epsilone의 약어로, 수학에서 작은 값을 나타내는데 사용, 컴퓨터에선 아주 작은 부동소수점값(미세오차 등에도 사용 됨)을 담는 변수의 이름으로 흔히 사용됨

      • 조건1 : 실제 데이터는 Variable의 인스턴스 변수인 data에 들어 있음

        def numerical_diff(f, x, eps=1e-4):
  x0 = Variable(x.data - eps)
  x1 = Variable(x.data + eps)
  y0 = f(x0)
  y1 = f(x1)
  return (y1.data - y0.data) / (2 * eps)

    2. Square 클래스를 대상으로 (수치)미분

      • $y = x^2$ 미분

      • $y = (e^{x^2})^2$ 미분

        def f(x):
   A = Square()
   B = Exp()

      • 일련의 계산을 f라는 함수로 정의한 뒤, 함수 f를 numerical_diff 함수에 전달함

        • 즉, 합성함수 f 가 미분 됨
        • 결론 : 아무리 복잡하게 조합된 함수라도 미분을 자동으로 계산할 수 있음!

    4.3 수치 미분의 문제점

    • 수치 미분의 결과에는 오차가 포함되어 있음

      • 대부분의 경우 오차는 매우 작지만, 어떤 계산이냐에 따라 커질 수 있음

      • 대표적인 이유

        자릿수 누락

      • 계산량이 많음

        • 변수가 여러 개인 계산을 미분할 경우 변수 각각을 미분해야 하기 때문
          • 신경망에선 매개변수를 수백만개 이상 사용하는 것은 기본이므로 수치미분을 사용하는 것은 현실적으로 사용 불가

    그러나 구현하기 쉽고, 값이 나름 정확하기 때문에 (그래도 역전파가 더 정확함), 역전파 구현 후 확인을 위해 수치 미분의 결과를 이용하기도 함. 이를 기울기 확인이라 함.

    기울기 확인(gradient checking) : 수치 미분 결과와 역전파의 결과 비교

    • [Step 10]에서 확인할 예정

    [Step 5] 역전파 이론

    역전파의 등장 : 계산 비용, 정확도 혜결

    • 미분의 효율적인 계산, 결과값의 오차를 더 줄임

    5.1 연쇄 법칙 (Chain rule)

    연쇄법칙

    • 합성 합수(여러 함수가 연결된 함수)의 미분은 구성 함수 각각을 미분한 후 곱한 것과 같다

      • 예시

        • 함수 $y=F(x)$ : 세 개의 함수 $a=A(x)$, $b=B(x)$, $c=C(x)$ 로 구성 됨

        • 계산 그래프

        • 식 : x에 대한 y의 미분

          • 구성 함수 각각의 미분값을 모두 곱한 값과 같음

          • 즉, 합성 함수의 미분은 각 함수의 국소적인 미분들로 분해가 가능함

            • [참고] $\frac{dy}{dy}$는 $y$에 대한 $y$의 미분(변화율)로 항상 1이다.

    5.2 역전파 원리 도출

    식 5.2에서, 합성 합수의 미분은 구성 함수 각각을 미분한 후 곱한 것과 같다는 것을 알았음

    곱하는 순서는 상관이 없기 때문에 출력에서 입력방향(역방향) 순으로 계산을 해보면 다음처럼 나타낼 수 있다.

    • 출력 쪽의 미분부터 순서대로 계산

    • $\frac{dy}{dx}$를 구하는 계산 그래프

    • 단순화한 역전파 계산 그래프

      • $A’(x)$의 곱셈을 ‘$A’(x)$’라는 노드로 간략하게 표현

      • $\frac{dy}{db}=C’(b)$, $\frac{db}{da}=B’(a)$, $\frac{da}{dx}=A’(x)$

        • $y$의 각 변수($y, b, a, x$)에 대한 미분값이 오른쪽으로 왼쪽으로 전파 됨

        • 전파되는 데이터는 모두 y의 미분값

        • $\frac{dy}{dy}, \frac{dy}{db}, \frac{db}{da}, \frac{da}{dx}$ 같은 $y$의 $\circ\circ$에 대한 미분값이 전파 됨

    왜 효율적일까?

    손실 함수의 출력을 얻을 때 계산효율이 좋음(이 값이 중요 요소임)

    • 머신러닝은 주로 대량의 매개변수를 입력 받아서 손실 함수(loss function)를 거쳐 출력을 내는 형태로 진행됨
      • 손실 함수의 출력은 단일한 스칼라 값인 경우 (많은 경우임)
        • 손실 함수의 각 매개변수에 대한 미분을 계산해야 함
        • 미분값을 출력에서 입력 방향으로 전파하면 한번의 전파만으로 모든 매개변수에 대한 미분을 계산할 수 있음

    5.3 계산 그래프로 살펴보기

    • 순전파 계산 그래프, 역전파 계산 그래프 비교

      |

      • 변수의 대응 (통상값 : 미분값)

        • 변수 $a$ : $\frac{dy}{da}$

        • 변수 $b$ : $\frac{dy}{db}$

        • 변수 $x$ : $\frac{dy}{dx}$

      • 함수의 대응 (통상 계산-순전파 : 미분값을 구하기 위한 계산-역전파)

        • 함수 $B$ : 역전파 $B’(a)$

        • 함수 $A$ : 역전파 $A’(x)$

      • 중요 : 역전파 시 순전파 시 이용한 데이터가 필요함

        • 함수 노드 $C’(b)$ : $y=C(b)$라는 계산의 미분

          • 계산시 $b$ 값이 필요하다

        • 함수 노드 $B’(a)$ : $y=B(a)$라는 계산의 미분

          • 계산시 $a$값이 필요하다

        각 함수가 입력변수($x,a,b$)의 값을 기억해야, 각 함수의 역전파를 구할 수 있음

    [Step 8] 재귀에서 반복문으로

    • 처리 효율 개선, 확장을 대비하여 backward 메서드 구현 방식 수정

      • 재귀 $\rightarrow$ 반복문

      • 재귀 함수는 호출 결과를 계속 메모리에 유지함 (스택에 쌓임) 따라서, 반복문 방식의 효율이 더 좋음

        • 메모리가 넉넉할 경우 사용하는 것은 문제가 되지 않으며, 꼬리 재귀(tail recursion) 기법을 이용하면 재귀를 반복문처럼 실행 할 수 있음
    import numpy as np

# 1
class Variable:
    def __init__(self, data):
        self.data = data
        # 6
        self.grad = None
        # 7
        self.creator = None
    # 7
    def set_creator(self, func):
        self.creator = func
    # 7 -> 8.2 (재귀 -> 반복문)
    def backward(self):
        funcs = [self.creator] 
        while funcs:           
            f = funcs.pop()    # 함수를 가져옴
            x, y = f.input, f.output # 함수의 입력과 출력을 가져옴  
            x.grad = f.backward(y.grad) # 자신보다 하나 앞에 놓인 변수의 backward 메서드 호출, 매개 변수로 출력(순전파의 결과)의 미분값을 넣어 줌
                                       

            if x.creator is not None:
                funcs.append(x.creator)  # creator가 계속 안만들어질 때까지 (순전파가 끝날 때까지) funcs 에 creator를 추가 함

    8.2 반복문을 이용한 구현 (Variable 클래스 backward 메소드 구현 방식 변경)

    • 재귀 $\rightarrow$ 반복문

      • 처리해야 할 함수들을 funcs 라는 리스트에 차례로 집어넣음

      • while 블록 안에서 처리해야 할 함수가 없을 때 까지, f의 input과 output(순전파의 입력값과 결과값)변수를 얻어서

      • f.backward()의 f의 output 변수에 미분값을 인수로 하면 그 입력값의 미분값을 구할 수 있다.

        class Variable:
   def __init__(self, data):
       self.data = data
       # 6
       self.grad = None
       # 7
       self.creator = None
   # 7
   def set_creator(self, func):
       self.creator = func
   # 7 -> 8.2 (재귀 -> 반복문)
   def backward(self):
       funcs = [self.creator] # (구버전 f=func) f = self.creator
       while funcs:           # (구버전 재귀문) if f is not None:
           f = funcs.pop()    # 함수를 가져옴
           x, y = f.input, f.output # 함수의 입력과 출력을 가져옴  # (구버전 입력만가져옴) x = f.input
           x.grad = f.backward(y.grad) # 자신보다 하나 앞에 놓인 변수의 backward 메서드 호출, 매개 변수로 출력(순전파의 결과)의 미분값을 넣어 줌
                                       # (구버전 - 자신보다 하나 앞에 놓인 변수의 backward 메서드 호출) x.grad = f.backward(self.grad)

           if x.creator is not None:
               funcs.append(x.creator) # 앞의 함수를 리스트에 추가한다. # creator가 계속 안만들어질 때까지 (순전파가 끝날 때까지) funcs 에 creator를 추가 함
