    Express 시작

    웹페이지 Express로 띄어보기

    왜 쓸까?

    1. 웹서비스 관점

      • Express 를 사용하지 않고 웹서버를 띄어볼 경우
        • 세션 처리 귀찮음 (http 내장모듈 사용, url 호출 등 해야함)

          Express 는 세션 처리 등 체계가 만들어져 있는 웹 프레임워크

        • 개발 방법론적 제약은 사라질 수 있으나, 그만큼 하나하나 세세하게 다 코딩해야함…

    2. 프레임워크 선정시 고려해야 될점

      사용자 수

      • Express 를 제일 많이 사용하므로 레퍼런스가 많으며, 유지보수가 쉬움
        • 판단 근거 : github 스타 수, …

    서버와 클라이언트

    서버(Server) : 서비스를 제공하는 쪽 <응답>

    클라이언트(Client) : 서비스를 제공받는 쪽 <요청>

    http 내장 모듈로 웹서버를 띄워보기

    내가 작성하는 응답 객체만 사용자가 볼 수 있음

    const http = require('http');

http.createServer( (request, response) => {  
    response.writeHead(200, {'Content-Type' : 'text/plain'});
    response.write('Hello Server');
    response.end();
}).listen(3000);

    1. request, response 를 인자로 받아서 작성

    2. 200 번 응답시(응답 성공시), 보낼 내용 작성

      [참고] http 상태코드
      상태코드 설명
      1xx 조건부응답
      2xx 응답성공
      3xx 리다이렉션
      4xx 요청오류(ex 404 Not Found)
      5xx 서버오류

    Routing

    url 추가

    • 라우팅 : 어떤 네트워크 안에서 통신 데이터를 보낼 때 최적의 경로를 선택하는 과정

    • express에서 라우팅 : 클라이언트로부터 요청받은 URL과 뷰를 매칭시키는 것

      • 즉, 특정한 URL에 대해 특정한 뷰로 연결하는 역할

    [예제] 텍스트 뿌리기

    GET /admin => admin 이후 url
    GET /admin/products => admin products

    1. route 폴더 만들기

    • 코드가 지저분해지지 않도록 분리하기 위함

    2. ./route/admin.js 만들고 코딩하기

    • 즉, 페이지 별로 분리
      • 여기선 admin으로 시작하는 페이지들(/admin, /admin/products, …)은 모두 admin.js에 코딩할 예정
    const express = require('express')
const router = express.Router();

router.get('/' , (req, res) => {
    res.send('admin 이후 url');
});

router.get('/products' , (req, res) => {
    res.send('admin products');
});

module.exports = router;

    3. app.js 에서 admin을 사용 코드 작성

    실행 할 파일에 (여기선 app.js)파일에 admin 을 require 해주고, express().use 해줌

    const express = require('express');

const admin = require('./routes/admin');

const app = express();
const port = 3000;

app.get('/', (req, res) => {
    res.send('hello express');
});

app.use('/admin', admin);

app.listen( port, () => {
    console.log('Express listening on port', port)
})

    • app.get 함수 : GET 방식으로 들어오는 경로를 연결하여, 이에 대응하는 동작을 콜백 함수로 기술할 수 있도록 함

      • 지정된 Path ‘/‘으로, 웹 사이트의 루트(root) 페이지로 연결

        • 루트 페이지 : 제일 처음 보이게 되는 메인 페이지

      • 콜백 함수의 req : 요청객체(Request Object)

        • 클라이언트에서 보낸 여러 정보가 포함

      • 콜백 함수의 res : 응답객체(Response Object)

        • 클라이언트에 응답할 수 있게 하는 객체로 위의 코드에선 send() 라는 함수로 메세지를 보내도록 작성됨

    즉, 위 코드는 설정된 경로의 루트 페이지(‘/‘)로 접속했을 때, ‘hello express’ 라는 메세지를 응답 받음

    app.js 작성 (express 관련 세팅)

    app.js 에선 express 관련 세팅만 (express 관련 모든 걸 여기 적기)

    클래스 화

    const express = require('express');
const nunjucks = require('nunjucks');
const logger = require('morgan')

class App {

    constructor () {
        this.app = express();

        // 뷰엔진 셋팅
        this.setViewEngine();

        // 미들웨어 셋팅
        this.setMiddleWare();

        // 정적 디렉토리 추가
        this.setStatic();

        // 로컬 변수
        this.setLocals();

        // 라우팅
        this.getRouting();

        // 404 페이지를 찾을수가 없음
        this.status404();

        // 에러처리
        this.errorHandler();

    }

    setMiddleWare (){

        // 미들웨어 셋팅
        this.app.use(logger('dev'));
        this.app.use(express.json());
        this.app.use(express.urlencoded({ extended: false }));

    }

    setViewEngine (){

        nunjucks.configure('template', {
            autoescape: true,
            express: this.app
        });

    }

    setStatic (){
        this.app.use('/uploads', express.static('uploads'));
    }

    setLocals(){

        // 템플릿 변수
        this.app.use( (req, res, next) => {
            this.app.locals.isLogin = true;
            this.app.locals.req_path = req.path;
            next();
        });

    }

    // 새로 추가
    getRouting (){
        this.app.use(require('./controllers'))
    }

    status404() {        
        this.app.use( ( req , res, _ ) => {
            res.status(404).render('common/404.html')
        });
    }

    errorHandler() {

        this.app.use( (err, req, res,  _ ) => {
            // console.log(err) // error 추적
            res.status(500).render('common/500.html')
        });

    }

}

module.exports = new App().app;
